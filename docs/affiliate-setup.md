## なぜやるのか（背景）

Kayoha のコア体験は「**通勤時間ベースで住む駅を決める**」。
これは日本の不動産 / 引越 / 通信業界が買う「**意思決定の瞬間**」のリードであり、
ユーザーの intent が高いのは事実。コード変更も最小限（駅詳細のボタンのみ）。

ただし「賃貸アフィリエイトで大きく稼げる」という初版の前提は **誇大だった**（下記 現実チェック）。
過度な期待を持たず、**主収益源ではなく「導線が既に出来ている低コストの受動収益層」** として
扱うのが現実的。現フェーズで注力すべきは広告チューニングではなく **トラフィック増**（SEO / 小紅書）。

## ⚠️ 現実チェック（2026-06-08 検証 — 旧計画の数値を訂正）

> 初版にあった「SUUMO ¥1,000/件 → 100 件で ¥10万」は **誤り**。
> 公開情報の多源クロスチェックで以下が判明（出典は本節末尾。正確な最新値は ASP 無料登録後の管理画面が確実）。

**1. 「SUUMO ¥1,000」は誤記の可能性が高い**
- ¥1,000 は実際には **スマイティ（価格.com 系、≠ SUUMO/Recruit）の問い合わせ単価**。初版が混同したと思われる。
- SUUMO 本体の「賃貸」案件は **終了している** との検証ブログ報告あり（単一ソース・要確認）。
- A8 での SUUMO 提携可否は **非公開**（メディア登録しないと不可視）。公開情報からは確認不能。

**2. 賃貸アフィリエイトの実単価（公開転載・参考値）**

| 成果点 | 実単価 |
|---|---|
| メール問い合わせ / 資料請求 | **¥300〜1,000**（¥1,000 は上限。¥300〜500 が常態）|
| 物件成約 | 数千〜¥10,000（ただし成約は極めて困難）|
| 不動産**投資**資料請求（≠賃貸・別客層）| ~¥20,000・承認率 ~80% |

**3. 賃貸は構造的に「稼ぎにくい」**
- 成果は **メール問い合わせのみ**カウント、**電話は対象外** → 急ぎの高 intent 客は電話するため成果化しない。
- 承認率が読めず非承認で報酬消滅。大手（SUUMO/HOME'S）は直接アクセスされ紹介経由は狭き門。1〜3 月以外は閑散。
- 実測例: ある運営者が 3 ASP・1 年で **月平均 ¥6,255**（お小遣い級）。

**4. 主要賃貸広告主は A8 より「バリューコマース」**
- DOOR賃貸 / HOME'S / SUUMO は **バリューコマース**経由が主（A8 ではない）。
- バリューコマースは **承認率を全員に公開** + DOOR賃貸は **即時提携** → 申請前に「白率」を見られる。
- 相対的に高単価の隣接案件: **HOME'S 住まいの窓口 ~¥12,500/件**（成果=相談予約）、**DOOR賃貸 お祝い金型**（ユーザーに最大 ¥10.3万）。

**5. 収益モデル（現実的な桁感）**

賃貸漏斗 = 訪問 × ボタン CTR(~2%) × 問い合わせ化(~1.5%) × 承認(~60%) × ¥500 → 訪問あたり ≈ **¥0.09**。
AdSense ≈ 訪問あたり ¥0.3（PV1.5 × RPM ¥200）だが、全画面地図は広告枠が乏しく UX を損なう。

| 月 UU | 賃貸 base | AdSense base |
|---|---|---|
| 5,000 | ¥450 | ¥1,500 |
| 20,000 | ¥1,800 | ¥6,000 |
| 50,000 | ¥4,500 | ¥15,000 |
| 200,000 | ¥18,000 | ¥60,000 |

→ **現トラフィック帯では両者とも小遣い級**。広告種別より **トラフィックが律速**。
GA4 の `affiliate_click` 実データで CTR を差し替えて精緻化すること。

## STEP 5：取得したリンクの管理（機微情報の取扱い）

提携承認後に各 ASP の管理画面で取得する **専用リンク URL** には個人のメディア ID が埋め込まれています（例: `a8mat=XXXXXX`）。
これは盗用されると報酬を横取りされる可能性があるため、以下の通り扱う：

- 🚫 このファイル（git 管理下）には書かない
- ✅ プロジェクトルートの `.env.local` に環境変数として置く

### `.env.local` テンプレ（コミット禁止）

```env
# 必須 — Supabase 接続
NEXT_PUBLIC_SUPABASE_URL=https://<project>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-key>

# A8.net media ID（提携承認後の数字を入れる。未設定なら fallback で各 ASP 通常 URL に遷移）
NEXT_PUBLIC_A8_MAT_SUUMO=
NEXT_PUBLIC_A8_MAT_HOMES=
NEXT_PUBLIC_A8_MAT_CHINTAI=

# 法務ページ運営者情報（未設定なら fallback 文字列が表示される）
OWNER_NAME=
OWNER_ADDRESS=
OWNER_PHONE=
CONTACT_EMAIL=
NEXT_PUBLIC_SITE_URL=https://kayoha.com
```

> `lib/affiliate.ts` の `wrapWithA8()` がメディア ID を**リテラル参照**で読み込み (Next.js の静的置換要件)、
> A8 リダイレクト URL `https://px.a8.net/svt/ejp?a8mat=...` を組み立てる。
> 未設定なら SUUMO / HOME'S / CHINTAI の通常 URL をそのまま開く。
