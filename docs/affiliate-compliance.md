## 0. なぜこれが必要か

アフィリエイト案件（特に SUUMO / HOME'S 等の不動産系）は **三重審査** が走る：

1. **景品表示法（ステマ規制）** — 消費者庁ガイドライン、2023-10-01 施行。違反は措置命令 + 公表
2. **ASP 個別規約** — A8.net / バリューコマース等。違反は提携解除 + 既往報酬没収
3. **広告主規約** — SUUMO（リクルート）/ HOME'S（LIFULL）等の個別 NG ワードリストあり

ASP 承認後はコードに手を入れる度に「合規敏感領域に触れていないか」を確認する必要がある。本ファイルはその確認の起点。

## 2. NG ワード辞書（UI 文案 / AI 出力 共通）

### 2-1. 絶対化表現（景表法・優良誤認の典型）

UI 上のボタンラベル・案内文 / AI 推薦の `reason` フィールド / メタタグ / OG description などに**絶対に入れないこと**：

| ❌ NG | 理由 | ✅ 置き換え例 |
|---|---|---|
| **最安** / 最も安い / 業界最低価格 | 価格優位の客観的根拠が必要、根拠なしは優良誤認 | 「家賃目安」「相場の参考値」 |
| **絶対** / 必ず / 100% | 確実性を保証する表現は景表法違反 | 「目安」「参考」「概ね」 |
| **保証** / 保障 | サービス内容を超える保証は誤認誘発 | 「推奨」「おすすめ」 |
| **完全** / 完璧 / 究極 | 客観的検証不能な最上級 | 削除 or「網羅的に」程度に弱化 |
| **必ず儲かる** / 損しない / お得確実 | 不動産は景表法上「資産性 / 値上がり」確約禁止 | 削除 |
| **公式** / 公認 / 提携 / 認定（SUUMO 公式等） | 商標 / ASP との関係を誇大表示 = 不正競争防止法も絡む | 「SUUMO で検索」程度の中立記述 |
| **業界 No.1** / 日本一 / 大人気 | 客観的根拠と調査期間明記が必要 | 削除 |

### 2-2. 不動産特有の NG ワード（広告主規約由来）

不動産業界には**おとり広告規制**＋**不動産公正競争規約** があり、ASP / 広告主から特に厳しく見られる：

| ❌ NG | 理由 |
|---|---|
| 「**理想的な物件**」「**最高の住環境**」 | 主観的優良表現、根拠なし |
| 「**新築同様**」（実際は中古の場合） | 不動産公正競争規約 |
| 「**駅から徒歩 5 分**」のような断定 | 当サイトは物件個別のデータを持たないので**断定しない**こと（駅単位までに留める） |
| 「**この駅は資産価値が上がる**」 | 投資勧誘扱い、宅建業法も絡む可能性 |
| 「**この街は治安が悪い**」 | 特定エリアの治安断定は名誉毀損 / 営業妨害リスク。「**治安良好**」も「**治安懸念**」も主観的断定は避ける |

### 2-3. ステマ規制特有の NG

| ❌ NG | 理由 |
|---|---|
| 「**個人的に SUUMO がおすすめ**」（PR 表記なし） | ステマ規制違反 |
| PR 表記を**目立たない位置・小さい文字・低コントラスト**で配置 | 消費者庁ガイドラインで「明瞭性」要件あり |
| affiliate link を**他の一般 link と見分けがつかない形式**で混入 | 同上 |

## 3. UI 合規敏感区（具体的な位置）

### 3-1. StationDrawer affiliate ボタン

**位置:** `components/StationDrawer.tsx` の L516-588 付近

**現状の合規実装（変更時は維持必須）:**
- ✅ セクションラベル「住居を探す」横に **「PR」マイクロラベル**（L536）
- ✅ ボタンは中立的な短縮ラベル: `SUUMO` / `HOME'S` / `CHINTAI`（`lib/affiliate.ts` の `ALL_PROGRAMS` 経由）
- ✅ `rel="noopener noreferrer sponsored"`（L545）— Google + 景表法推奨
- ✅ `target="_blank"`（L544）— 利用者に「外部サイトに遷移する」明示
- ✅ ボタン直下に**「広告について」link**（L584、`/legal/ads` へ遷移）
- ✅ アフィリエイトセクションの直前に**通勤時間 ±5〜10 分の免責文**（L510 周辺）

**改修時の注意:**
- 「PR」ラベルは**ボタンと視覚的に同一視野に入る位置**に維持。離す / 隠す / 折りたたみ内に入れる禁止
- ボタンラベルを「**おすすめ物件を見る**」「**最安物件**」のような誘導的表現に変更禁止
- `rel="sponsored"` を**削除しない**（SEO 上も景表法上も推奨）
- `/legal/ads` への link を削除しない / `display:none` 禁止

### 3-2. AI 推薦結果のテキスト

**位置:**
- `components/AiResultGrid.tsx`（カード表示側）
- `lib/ai-recommend/prompt.ts`（AI に渡すガード文言）

**現状の合規実装:**
- ✅ system prompt（`prompt.ts` L10-33）に下記が明記済み:
  - 「固有施設名（特定の店・ライブハウス・会社名）の幻覚は絶対禁止」
  - 「素晴らしい街で皆さんに大人気！」のような**宣伝口調禁止**
  - editorial 調・80 文字以内
- ✅ 「特定民族・文化圏向けの表現は避け、誰にでも通じるニュートラルな書き方」（多言語化対応 + 差別表現リスク回避）

**改修時の注意:**
- prompt の合規ガード文言を**削除・弱化しないこと**
- 推奨：§8 に列挙する**絶対化表現禁止ガード**を順次追加していく
- AI 出力 `reason` 文字列を UI 側でそのまま表示する設計なので、prompt の品質 = UI の合規性に直結

### 3-3. AI Wizard の destination / 偏好設問

**位置:** `components/AiWizard.tsx`

**改修時の注意:**
- 偏好の選択肢ラベル（「治安重視度」「街の雰囲気」など）に**断定的価値判断**を入れない
  - ❌「治安悪いエリアを避ける」 → ✅「治安重視」「治安を最優先」
- destination 候補の表示で「**この駅は人気**」「**おすすめ**」等を勝手に追加しない

### 3-4. PR 表記の「明瞭性」要件

消費者庁ステマ規制ガイドライン上の明瞭性要件は以下のすべてを満たすこと：

- [ ] PR 表記の**文字サイズ**が周辺の通常テキストと同等以上、もしくは判読可能
- [ ] PR 表記の**色 / 背景コントラスト**が WCAG AA 程度（背景に紛れない）
- [ ] PR 表記とアフィリエイト link の**視覚的距離**が近い（同一視野内）
- [ ] スクロール / アコーディオン / モーダルで PR 表記だけが隠れる構造ではない
- [ ] モバイル表示でも上記が崩れない

現状の StationDrawer 実装は L524-537 で `padding 1px 6px`・`fontSize 10`・`fontWeight 700`・`background rgba(28,24,18,.08)` で表示しており、リファインメント時は上記要件をクリアしているか確認する。

## 5. レッドライン行為（一発提携解除）

ASP / 広告主が**最も厳しく取り締まる**行為：

### 5-1. 自己契約 / 関係者契約による報酬詐取

- ❌ 自分のアフィリエイト link 経由で自分が SUUMO に物件問い合わせ → 報酬発生
- ❌ 家族 / 友人 / 同居人に「私の link 経由で申し込んで」と依頼
- ❌ 同一 IP / 同一決済情報での成約

**回避策:** ASP 承認後、自分自身が SUUMO で物件を探す際は **当サイトを経由せず直接 `suumo.jp` にアクセス**。家族にも同様の依頼を絶対にしない。

### 5-2. クリック詐欺 / コンバージョン詐欺

- ❌ Bot / スクリプトで affiliate link をクリック
- ❌ Cookie stuffing（利用者が link を踏まなくても cookie を強制セット）
- ❌ iframe / `window.open` / リダイレクトで利用者意図なく affiliate URL を踏ませる
- ❌ ポップアップ / ポップアンダーで強制遷移

**現状の実装は安全:** `StationDrawer.tsx` の affiliate ボタンは **利用者の能動的クリック** のみで遷移する `<a>` タグ。これを変更しないこと。

### 5-3. 商標 / 信用毀損

- ❌ SUUMO / HOME'S / CHINTAI のロゴを**正規許諾なく**使用
- ❌ 「SUUMO 公式」「HOME'S 認定パートナー」等の虚偽記載
- ❌ 競合他社（楽待・健美家等）への中傷

**回避策:** ロゴは使わずテキストラベル（現状の実装通り）。広告主名は事実記載のみに留める。

### 5-4. ステマ

- ❌ PR 表記を**削除 or 隠蔽** した状態で affiliate link を配置
- ❌ 「個人の感想です」を装って広告主商品を推奨
- ❌ AI 推薦テキスト中で affiliate 広告主の優位性を誇大表現

## 6. 重大変更時の事前確認フロー

下記を実施・検討する前には **本ファイルを更新 + ASP / 広告主への事前相談を検討**：

| 変更内容 | リスク | 事前確認 |
|---|---|---|
| **独自ドメイン取得・切り替え** | ASP は登録ドメインに紐づくため、切替で再審査必要なケースあり | A8.net マイページから「サイト追加 / 変更」申請 |
| **AdSense / 他広告ネットワーク併用** | ASP / 広告主によっては競合プログラム同時掲載 NG の規約あり | 各 ASP の規約条項を確認 |
| **付費 / サブスク機能追加** | サービス性質が変わるため再審査の可能性 | ASP 担当に事前相談 |
| **多言語化（英語 / 中国語版公開）** | 言語別 PR 表記・各国法令対応が必要 | 各言語版でも `/legal/ads` 等翻訳必須 |
| **SUUMO / HOME'S 以外の不動産系 affiliate 追加** | 既存広告主との利益相反条項あり得る | 既存広告主の規約確認 |
| **ユーザーレビュー / コメント機能拡張** | 「ステマ依頼」の温床になり得るため運用ポリシー必須 | 投稿ガイドライン + モデレーション体制設計 |
| **メール配信 / プッシュ通知に affiliate link 同梱** | 規約上禁止する ASP が多い | 各 ASP の規約確認 |

## 8. AI 推薦 prompt の合規ガード（推奨追加）

現在の `SYSTEM_PROMPT` にすでに以下は含まれている：
- ✅ 固有施設名禁止
- ✅ 宣伝口調禁止
- ✅ ニュートラル表現要求

**追加検討推奨：**

```text
# 合規ガード（追加候補）
- 「最安」「絶対」「必ず」「保証」等の絶対化表現禁止
- 「公式」「公認」「認定」等の権威付け表現禁止
- 「この駅は資産価値が上がる」等の投資勧誘的表現禁止
- 「治安悪い」「危ない」等の特定エリアへの断定的否定評価禁止
- 「治安良好」「安全」等も主観的断定として避け、「ファミリー層多め」等の客観的記述に置換
- 物件個別の情報（家賃・徒歩分数の断定）は出力しない（当サイトは駅単位のデータしか持たない）
```

これを `prompt.ts` の `SYSTEM_PROMPT` に追加する場合、`# 出力ルール` セクションの末尾に挿入する。AI 出力品質と合規を同時に強化できる。

> **未実装の理由（2026-05-14 時点）:** 主人と本小姐の判断で、まず ASP 申請を通すことを優先し、prompt の細かい合規強化はリリース後に対応する方針。本ファイル §8 は「次に強化するならここ」というメモとして機能する。

## 9. インシデント対応

ASP / 広告主 / 消費者庁 / 利用者から合規違反指摘を受けた場合：

1. **24 時間以内に該当箇所を一時的に非表示** — affiliate ボタンを `display:none` 等で隠す
2. 指摘内容を本ファイルの改訂履歴に記録
3. 修正後、§7 自己チェックを全項目クリアしてから再公開
4. ASP / 広告主には誠実に状況報告（再発防止策込み）
